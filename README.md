# 寻宠平台（项目代号 petfinder）

> 本地化宠物走失/招领信息互助小程序，基于"善意网络"理念。
> 一句话定位：**让每一份善意，都能找到回家的路**。

---

## 项目状态

- **阶段**：MVP 设计阶段（需求初稿 + 核心机制讨论完成）
- **目标**：单城市启动，2 个月内完成 MVP 并验证需求
- **技术栈**：微信小程序 + 云开发（云函数 + 云数据库 + 云存储）
- **运营**：开发者全职 + 2-3 名志愿者
- **变现路径**：MVP 不变现，V2 探索抽成/会员/B 端合作（公益外衣 + 谨慎变现）

## 与原始需求 V2.0 的关键差异

本项目设计基于一份原始需求文档 V2.0，**有以下重要调整**：

1. **善行值激励重排** — 原方案"线索 +1 vs 归还 +6"激励反向，新方案改为线索基础奖励 +2 + 关键线索追加 +3，重新匹配"行为成本 vs 网络贡献"
2. **新增 24h 归还冷却期** — 双方确认归还后冷却 24h 再结算善行值，作为冒领的最后防线
3. **取消"爬取微博/小红书"预填充** — 改为人工汇总公开寻宠群信息 + 主动联系失主授权代发，规避法律和审核风险
4. **善行值在 MVP 加月度榜单** — 不能光攒数字，需要仪式感激励持续行为

详见 `docs/00-overview.md`。
