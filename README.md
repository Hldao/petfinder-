**v5 包含的 16 屏**：首页信息流 / 地图视图 / 寻宠详情 / 招领详情 / 发布表单 / 联系失主沟通建议 / 归还自检 / 归还成功 / 感谢信编辑 / 消息列表 / 消息对话 / 个人中心 / 善行值流水 / 我的发布 / 争议申诉 / 后台审核（注释区有跳转网格可 debug 任一屏）

**v6 相对 v5 的变化**：归还流程从 4 屏砍到 3 屏；checklist 砍掉，改为一键归还；"沟通建议"从主流程独立屏降级为按需 modal（在详情页和归还确认页都可调出）。详见 `docs/06-product-philosophy.md` § "决策演进轨迹"。

> ⚠️ 当前原型 (v1-v6) 里使用的善行值数字（线索 +1 / 归还 +6）沿用 V2.0 老规则，与 `docs/01-points-system.md` 里二轮调优后的新规则（线索 +2 + 关键 +3 / 临时照料 +5 / 归还 +4）**尚未对齐**。下次原型迭代或正式开发时统一更新。

## 与原始需求 V2.0 的关键差异

本项目设计基于一份原始需求文档 V2.0，**有以下重要调整**：

1. **善行值激励重排** — 原方案"线索 +1 vs 归还 +6"激励反向，新方案改为线索基础奖励 +2 + 关键线索追加 +3，重新匹配"行为成本 vs 网络贡献"
2. **新增 24h 归还冷却期** — 双方确认归还后冷却 24h 再结算善行值，作为冒领的最后防线
3. **取消"爬取微博/小红书"预填充** — 改为人工汇总公开寻宠群信息 + 主动联系失主授权代发，规避法律和审核风险
4. **善行值在 MVP 加月度榜单** — 不能光攒数字，需要仪式感激励持续行为

详见 `docs/00-overview.md`。
