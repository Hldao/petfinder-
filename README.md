# 寻宠平台（项目代号 petfinder）

> 本地化宠物走失/招领信息互助小程序，基于"善意网络"理念。
> 一句话定位：**让每一份善意，都能找到回家的路**。

**v5 包含的 16 屏**：首页信息流 / 地图视图 / 寻宠详情 / 招领详情 / 发布表单 / 联系失主沟通建议 / 归还自检 / 归还成功 / 感谢信编辑 / 消息列表 / 消息对话 / 个人中心 / 善行值流水 / 我的发布 / 争议申诉 / 后台审核

**v6 相对 v5 的变化**：归还流程从 4 屏砍到 3 屏；checklist 砍掉，改为一键归还；"沟通建议"从主流程独立屏降级为按需 modal。详见 `docs/06-product-philosophy.md` § "决策演进轨迹"。

**v7 → v8 的变化**（视觉系统重做 · 设计语言从"沉重"转向"治愈"）：

- 🎨 **配色** · 板蓝根靛青/朱砂红 → **杏橘 #FF8552 / 薄荷 #5BB89F / 蜜糖 #F0AC32 / 紫粉 #A672D4**
- 🎨 **字体** · 宋体/思源宋体 → **苹方 / Misans + Manrope**（圆润现代，摆脱衬线沉重感）
- 🎨 **形态** · 印章/邮戳/纸纹/邮票虚线 → **大圆角 + 柔和阴影 + 圆胶囊状态徽章**
- 🎨 **板块分隔** · 三层视觉层级（L0 屏幕底 #F0E9D8 / L1 卡片纯白 / L2 悬浮卡片更深阴影）
- 🎨 **设计意图** · "明信片仪式感" → "**让焦虑的人松一口气**" — 用户打开 app 时本就在难过，UI 不应叠加心理负担

**v7 的合规改造在 v8 完整保留**（详见 `docs/05-risks.md`）：admin 剥离 · 站内消息/虚拟号 · 发布须知禁悬赏 · 个人中心补全协议/客服/注销 · 善行值不可兑换 · 聊天审核提示 · 启动隐私 Modal · 位置精度模糊。

> 本轮 v8 **不做动效**（聚焦视觉），后续可叠加悬浮、加载、状态切换等微交互。

> ⚠️ 当前原型 (v1-v6) 里使用的善行值数字（线索 +1 / 归还 +6）沿用 V2.0 老规则，与 `docs/01-points-system.md` 里二轮调优后的新规则（线索 +2 + 关键 +3 / 临时照料 +5 / 归还 +4）**尚未对齐**。下次原型迭代或正式开发时统一更新。

## 与原始需求 V2.0 的关键差异

本项目设计基于一份原始需求文档 V2.0，**有以下重要调整**：

1. **善行值激励重排** — 原方案"线索 +1 vs 归还 +6"激励反向，新方案改为线索基础奖励 +2 + 关键线索追加 +3，重新匹配"行为成本 vs 网络贡献"
2. **新增 24h 归还冷却期** — 双方确认归还后冷却 24h 再结算善行值，作为冒领的最后防线
3. **取消"爬取微博/小红书"预填充** — 改为人工汇总公开寻宠群信息 + 主动联系失主授权代发，规避法律和审核风险
4. **善行值在 MVP 加月度榜单** — 不能光攒数字，需要仪式感激励持续行为

详见 `docs/00-overview.md`。
