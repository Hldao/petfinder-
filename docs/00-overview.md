# 00 — 项目概览

## 定位

本地化宠物走失/招领信息互助平台，**基于"善意网络"理念**。

### 三条底线（不可破）

1. 平台不介入任何金钱交易（悬赏、感谢金等由用户自行线下）
2. 善行值仅作为荣誉记录，MVP 阶段不实现兑换或捐赠
3. 单城市启动，由开发者亲自运营 + 志愿者辅助

### 目标定位

- 阶段：**创业项目验证**
- MVP 阶段：完全不变现，验证需求与社区粘性
- V2 探索：抽成寻宠师服务 / 救助组织合作 / B 端会员（与"善意"定位保持兼容）

---

## 用户角色（行为定义，不预设）

| 角色 | 典型行为 |
|------|---------|
| 失主 | 发布寻宠、标记线索、确认归还、写感谢信 |
| 发现者 | 发布招领、临时照料、确认归还 |
| 热心人 | 提供线索、举报虚假 |
| 守护者（自动） | 累计善行值高的用户，获徽章 |

---

## 核心业务流程

### 寻宠流程
```
失主发布 → 系统生成海报 → 他人提供线索 → 失主标记"有用" → 找回 → 双方确认归还 → 24h 冷却 → 善行值结算 → 失主写感谢信
```

### 招领流程
```
发现者发布（可勾选临时照料） → 失主看到并联系 → 双方核对私有特征（视频/线下） → 线下交接 → 双方确认归还 → 24h 冷却 → 善行值结算
```

---

## 关键设计决策

### 1. 善行值激励重排（详见 01）

原方案"线索 +1 / 归还 +6"激励反向，导致线索池稀薄风险。新方案按"行为成本 + 网络贡献"双维度定价。

### 2. 24h 冷却期（详见 02）

双方确认归还后善行值不立即结算，留 24 小时反悔窗口。这是冒领的最后一道防线。

### 3. 防冒领的真实边界

私有特征只防陌生人冒领，防不住"代答 / 转述"。真实防御靠**线下交接 + 视频通话 + 疫苗本/旧照核验**。平台只能强引导，不能强保证。

### 4. 状态机三表（详见 02）

`lost_pets / found_pets / return_confirmations` 各有独立状态机，互相耦合的转换点必须显式写出来。

### 5. 一只宠物多人匹配

一只走失宠物可能被多人误认/捡到。`return_confirmations` 设计为 1:N，但只有一个能进 `confirmed`。

### 6. 数据来源合规

不爬取微博/小红书。冷启动数据来源：
- 开发者人工汇总本地寻宠群公开信息
- **必须**联系原发布者获得授权代发
- 标注"来源：XX 群 / 已获授权"
