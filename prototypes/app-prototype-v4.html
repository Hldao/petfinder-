🎯 v4 的产品哲学定稿

  • 平台 = 撮合 + 关键节点 + 激励。不是留痕系统，不是流程管控者
  • 关键节点必须在平台内：发布、认领进入、归还确认、感谢信、申诉、善行值。其他都在外：聊天、约见面、私加微信、线下核对
  • "私有特征"字段：定位为认知工具 + 争议仲裁记录，不是物理阻断
  • 用户行为自由：复制话术、用什么沟通、是否勾选自检、是否在平台留言 — 都不强求

🔒 几条不能放松的合规底线

  • UI 上不出现"加微信 / 留微信号"字样（微信审核硬红线）
  • 平台留言系统仍要做（首次接触必需）但定位为"启动器"，不是"主战场"
  • 关键节点必须在小程序内（这是平台能"看到归还"的唯一方式，也是发善行值的依据）
📲 完整可点击路径：
首页 → 橙猫卡片 → 招领详情 → 这是我家的猫 → 联系失主（话术 + 安全地点 + 见面建议）→ 我已联系上失主 → 归还自检（点 checklist 项切换）→ 确认归还 → 归还成功
