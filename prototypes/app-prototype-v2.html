🆕 信息流的三处改动

  • 排序切换：橙色"⏱ 最新 ▾"chip 放在筛选条最右端，点击弹出底部 popup 让用户切换"最新发布 / 距离最近 / 互动最多"（V2 才上）。默认值是"最新"——冷启动期数据少，时间排序让社区有活跃感
  • 时间分级：< 1 小时显示 ⚡ 10 分钟前（橙色紧迫）；< 24h 显示"x 小时前"（深色）；> 24h 显示"昨天 16:00"（中灰）；> 7 天显示"9 天前 · 信息可能过期"
  • 过期淡化：> 7 天的卡片整体降饱和度 + opacity 0.55，让"陈旧但还在的"信息有适度可见性，但不抢占新内容注意力

📝 发布表单的关键设计

  • 顶部段选 "我丢了 / 我捡到"：两种表单字段差异在 30% 以内，没必要分两个入口；段选切换更轻量
  • 私有特征独立成"黄色卡片"：从普通表单字段里拎出来视觉强调，让用户意识到"这一段不公开、是用于核对的"——直接对应你 V2.0 防冒领的核心字段
  • 联系方式默认"仅平台留言"：保护用户隐私是默认行为，公开手机号是用户主动选择
  • 悬赏 / 感谢金提示：放在表单底部的灰色小字，明确"平台不介入金钱交易"——直接呼应 V2.0 的法律风险应对

🤝 归还确认流程的关键设计

  • 认领核对（屏 5）：失主填的"私有特征"由发现者反向核对 — UI 上引导发现者描述"只有真正失主才知道的细节"。配合红色冒领警告，让恶意成本透明可见
  • 归还 checklist（屏 6）：5 项打勾——3 项系统自动标记完成（特征核对、视频核对、约定交接），2 项需双方各自勾选（亲自接回 / 亲手交付）。"等待双方确认（3/5）"的状态提示让进度感清晰
  • 归还成功（屏 7）：绿色 hero circle + 橙色奖励卡 + 感谢信 CTA 三段式。"会出现在 / 不会出现在"清单透明告诉用户隐私边界，是反向"善意社区"承诺的可见呈现
